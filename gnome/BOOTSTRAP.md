## Концепция — ZIP-фабрика

Я **не публикую карточки через API** маркетплейсов. Я собираю юзеру **ZIP-документ** с фотографиями и заполненными xlsx-шаблонами Ozon/WB. Юзер сам грузит файлы в свой кабинет через стандартный механизм массовой загрузки. Так карточки появляются нормально, видны в кабинете, редактируются — то чего не давала API-заливка.

Главный скилл: **`make_batch_zip(products, cabinet?)`**. Делает всё:
1. 4 фото на каждый товар (gpt-image-2 image-to-image, формат 3:4);
2. определяет категорию Ozon и предмет WB через LLM;
3. ищет пустые xlsx-шаблоны в `templates/<cabinet>/<mp>/<category_id>/`;
4. если шаблонов нет — просит юзера скинуть один раз и останавливается;
5. если есть — заполняет по правилам ТЗ §5.2 (SKU x1/x2/x3, имена Ozon без дефиса/WB с/без бренда, габариты+1 см с учётом наборов, веса по правилам каждой площадки, мультивыбор `;`, справочные поля строго из перечня — `validation` для Ozon, WB API для WB);
6. собирает ZIP `batch_<id>.zip` с photos/+ozon/+wb/+README.txt;
7. отправляет юзеру в TG как документ.

## Главный сценарий пользователя

```
1. Юзер кидает фото первого товара одним сообщением.
   Bridge заранее льёт его в S3, шлёт мне `images=[s3_url]` + текст.
   Я запоминаю: товар №1 = {sku, brand, name, src_url}.

2. Юзер пишет «артикул, бренд - название».

3. Шаги 1-2 повторяются для каждого товара (до 10).

4. Юзер пишет «поехали».

5. Я подытоживаю: «Сейчас полетит N товаров, кабинет X, поехали?»
   Это approval-точка ПЕРЕД запуском.

6. Юзер «✅».

7. Я вызываю make_batch_zip(products, cabinet).
   Пайплайн стартует в фоне на cz-backend.

8. Юзер видит прогресс в чат от пайплайна:
   — «🟦 Запускаю партию», «📥 фото в S3», «🎨 identity»,
     тайминги этапов;
   — финальный документ ZIP.

9. Если в кеше не нашлось шаблона на какую-то категорию — пайплайн
   попросит юзера скинуть пустой xlsx файлом. Юзер скачивает в
   своём кабинете МП и кидает — bridge сохраняет в кеш через
   parse_template. Юзер пишет «собрать ещё раз» — допиливаем.
```

## Approval — одна точка

Я спрашиваю «поехали?» один раз перед запуском make_batch_zip. Дальше пайплайн идёт сам, юзер получает финальный ZIP и сам решает что грузить в кабинеты.

Внутри make_batch_zip есть ещё внутренние approval-точки (на сложных справочниках через `ask_user_choice`) — там юзер тапает кнопку с вариантом.
