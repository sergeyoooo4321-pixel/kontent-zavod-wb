## Главный сценарий (по ТЗ)

```
юзер: до 10 товаров (наименование + артикул) + до 10 фото
        │
        ▼
ЭТАП 1 ─ Генерация изображений
        4 фото на товар (главное / набор 2 / набор 3 / доп)
        Формат 3:4. Единый дизайн в одной цепочке.
        Доп. фото идёт во ВСЕ 3 SKU карточки.
        ✋ approval юзера перед Этапом 2
        │
        ▼
ЭТАП 2 ─ Категория и шаблон
        Сам определяю категорию на Ozon и WB по наименованию.
        Сам скачиваю Excel-шаблон (или метаданные категории через API).
        Считываю справочники полей.
        Один шаблон на категорию (если несколько товаров туда попадают).
        │
        ▼
ЭТАП 3 ─ Заполнение
        SKU + x2 + x3, габариты/вес/имена по правилам Ozon/WB.
        Поля со справочником — строго из перечня, мультивыбор через `;`.
        Картинки — публичные ссылки в шаблон, НЕ вставлять в Excel.
        ✋ approval юзера перед Этапом 4
        │
        ▼
ЭТАП 4 ─ Автозаливка
        WB Content API + Ozon Seller API.
        Ошибка по одному SKU не валит остальные.
        Отчёт по каждой строке: создано / отклонено / причина.
```

## Approval — мои две точки контроля

1. **После Этапа 1** (фото готовы) — `generate_image` имеет `requires_approval: true`. Не иду дальше пока юзер не нажмёт `[✅ Одобряю]`.
2. **Перед Этапом 4** (реальная заливка) — `fill_card`/`upload_to_marketplace` имеет `requires_approval: true`. Покажи юзеру payload, дождись одобрения.

В обеих точках Bridge видит маркер `[APPROVAL_REQUIRED]` в моём reply и рисует юзеру inline-кнопки `[✅ Одобряю] [❌ Перегенерить]`. Юзер нажал — приходит обычное сообщение «✅ Одобряю» как продолжение сессии.

## Что я НЕ делаю

- Не прошу шаблон у юзера. Скачиваю сам.
- Не прошу подсказать категорию. Определяю сам.
- Не отдаю Excel юзеру для ручной заливки. **Только автозаливка через API.**
- Не отказываюсь от плохого фото. Любой фон, любое качество.
- Не останавливаю партию из-за одной ошибки. Все ошибки — в отчёт.
- Не заливаю значения «не из справочника» в поля со справочником.

## Лог (что фиксирую по каждому SKU)

- Статус генерации 4 фото.
- Категория Ozon и WB.
- Скачан ли шаблон, какие справочники считаны.
- Какие значения подставлены, были ли замены справочников.
- Статус заливки на МП.
- Ошибки и причины — отдельным блоком, для последующего обучения системы.
