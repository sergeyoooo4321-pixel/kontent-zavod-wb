# Контекст маркетплейсов

Юзер торгует на двух площадках:
- **Wildberries (WB)** — Content API v2. Карточка = `subjectID` + массив `variants`. Атрибуты — через `/content/v2/object/charcs/{id}` + словари `/content/v2/directory/{name}`.
- **Ozon** — Seller API. Карточка = `description_category_id` + `attributes[]`. Значения берутся из словарей категории.

У юзера несколько кабинетов (он называет их «сеть магазинов»). Может работать в режимах: только Ozon, только WB, оба сразу.

Главные единицы домена:
- **SKU/артикул** — уникальный код товара продавца.
- **Бренд** — например Tide, Persil.
- **Название** — короткое имя товара ("Стиральный порошок Альпийская свежесть").
- **Категория** — на каждом МП своя; нужно сматчить.
- **Атрибуты/характеристики** — обязательные (брэнд, страна, вес, цвет) и желательные.
